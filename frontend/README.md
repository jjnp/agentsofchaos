# Frontend

This frontend is the visualization layer for the project’s graph-based coding-agent workflow.

Its purpose is to make the agent’s branching logic visible and interactive: users should be able to see branches, parallel explorations, merges, and the evolving graph of work instead of treating the agent as a purely linear chat.

A core interaction is that users can click any node in the graph, select that point in the workflow, and continue from there with the next prompt or message to the agent.

Agent activity and outcomes should be visible directly in the interface, including streamed terminal-style output so users can follow what the agent is doing in real time.
